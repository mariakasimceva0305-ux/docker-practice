# Контейнеризированные сервисы вычисления функций потерь

## Кратко
Репозиторий показывает раздельный запуск сервисов вычисления функций потерь MSE, MAE и squared hinge loss через Docker Compose. Основная идея — вынести отдельные вычислительные функции в независимые контейнеры с единым воспроизводимым запуском.

## Задача
Продемонстрировать базовый шаблон контейнеризации нескольких независимых Python-сервисов, каждый из которых отвечает за свою функцию потерь и может запускаться в общем контуре.

## Архитектура
```text
[входные данные]
      |
      +----------------+----------------+----------------+
      |                |                |
      v                v                v
 [сервис MSE]     [сервис MAE]   [сервис squared hinge loss]
      |                |                |
      +----------------+----------------+
                       |
                       v
             [общий запуск через Docker Compose]
```

## Структура репозитория
```text
src/               исходный код сервисов
Dockerfile         контейнерный образ
docker-compose.yml общий запуск сервисов
```

## Что показывает проект
- раздельную упаковку вычислительной логики по сервисам;
- единый воспроизводимый запуск через Docker Compose;
- базовый паттерн организации небольших вычислительных сервисов.

## Запуск
```bash
docker compose up --build
```

## Ограничения
- проект не включает оркестрацию, мониторинг и промышленное логирование;
- набор функций ограничен демонстрационным контуром;
- для производственного использования нужны API-контракты, тесты нагрузки и обработка ошибок.
